# 象棋规则实现说明

## 已实现的规则

根据你的描述，我实现了以下象棋规则：

### 1. 单方长将（一方连续将军）
- **规则**：必须变着
- **实现**：连续将军6次判为长将，强制要求变着
- **检测方法**：`isOneSidePerpetualCheck()`

### 2. 单方长捉（追捉同一子）
- **规则**：必须变着
- **实现**：连续攻击同一棋子3次判为长捉，强制要求变着
- **检测方法**：`isOneSidePerpetualAttack()`

### 3. 双方长将 / 互相长捉
- **规则**：双方不变作和，判和（双方责任对等）
- **实现**：
  - 双方长将：双方各连续将军3次，提示和棋
  - 双方长捉：双方各连续攻击2次，提示和棋
- **检测方法**：`isBothSidesPerpetualCheck()`, `isBothSidesPerpetualAttack()`

### 4. 双方均闲着（无攻击意图）[已移除]
- **规则**：此功能已移除，不再提示和棋

### 5. 一方禁止，一方允许
- **规则**：禁止方必须变着，不变判负，禁止方判负
- **实现**：
  - 一方有长将或长捉（禁止着法），另一方没有
  - 要求禁止方变着，否则判负
- **检测方法**：`isOneForbiddenOneAllowed()`, `getForbiddenSide()`

## 其他已实现的规则

### 1. 三次重复局面
- **规则**：三次重复局面判和
- **实现**：同一局面（包含回合信息）出现3次，强制变着

### 2. 30回合未吃子
- **规则**：双方30回合内未吃子判和
- **实现**：`peaceRound >= 30`时提示和棋

### 3. 双方无攻击性棋子
- **规则**：双方都无攻击性棋子（车、马、炮、兵/卒）判和
- **实现**：`attackNum_B == 0 && attackNum_R == 0`时提示和棋

## 技术实现细节

### 1. 长将检测
- 使用`consecutiveCheckRed`和`consecutiveCheckBlack`计数器
- 每次将军增加对应方计数器，重置对方计数器
- 非将军时重置当前行棋方计数器

### 2. 长捉检测
- 使用`consecutiveAttackRed`和`consecutiveAttackBlack`计数器
- 记录被攻击棋子的位置和类型(`lastAttackedPiecePos`, `lastAttackedPieceType`)
- 连续攻击同一棋子时增加计数器
- 攻击不同棋子或吃子时重置计数器

### 3. 攻击性棋子判断
- 黑方攻击棋子：车(5)、马(4)、炮(6)、卒(7)
- 红方攻击棋子：车(12)、马(11)、炮(13)、兵(14)

### 4. 规则优先级
1. 三次重复局面
2. 长将/长捉规则
3. 其他和棋条件（30回合未吃子、无攻击性棋子等）

## 测试建议

1. **长将测试**：让一方连续将军6次，观察是否提示强制变着
2. **长捉测试**：让一方连续攻击同一棋子3次，观察是否提示强制变着
3. **双方长将测试**：双方轮流将军各3次，观察是否提示和棋
4. **一方禁止测试**：一方长将，另一方走闲棋，观察是否要求长将方变着

## 注意事项

1. 长捉检测可能需要根据实际对局情况进行调整阈值
2. 攻击意图判断基于棋子移动后的可能攻击目标
3. 规则优先级确保了重要的规则（如长将）优先处理
4. 用户界面提示需要清晰明了，避免混淆
