# 棋局保存逻辑

## 1. 保存流程

### 1.1 调用入口
- 在 `PvMActivity` 中，用户点击保存按钮后调用 `saveChessNotation` 方法
- 方法签名：`private void saveChessNotation(String fileName, String redPlayer, String blackPlayer, String date, String location) throws Exception`

### 1.2 棋谱对象创建与信息设置
1. 创建 `ChessNotation` 对象
2. 设置基本信息：
   - 文件名
   - 当前日期
   - 红方玩家名称
   - 黑方玩家名称
   - 对局日期
   - 对局地点

### 1.3 FEN 信息生成
- 调用 `generateFEN` 方法生成当前棋盘状态的 FEN 字符串
- FEN 格式：
  ```
  [棋盘部分] [回合] - - 0 1
  ```
  - 棋盘部分：从红方底线开始（y=0）到黑方底线结束（y=9）
  - 回合：当前轮到哪方走棋（红方 'b'，黑方 'w'）
  - castle 部分：中国象棋不需要，设为 "- -"
  - 半回合计数和全回合计数：设为 0 1

### 1.4 走法记录提取
1. 从 `infoSet.preInfo` 栈中提取走法：
   - 临时栈保存弹出的 `ChessInfo` 对象
   - 对每个 `ChessInfo` 对象，生成走法字符串
   - 红方走法：添加新记录
   - 黑方走法：更新最后一条记录
   - 恢复 `infoSet.preInfo` 栈

2. 处理当前 `chessInfo` 中的最后一步走法：
   - 同样生成走法字符串
   - 根据颜色添加或更新记录

### 1.5 文件保存
- 调用 `notation.saveToFile(this, fileName)` 保存棋谱
- `saveToFile` 方法：
  - 清理文件名（去除后缀）
  - 调用 `SaveInfo.SerializeChessNotation` 序列化
  - 使用 `.pgn` 扩展名

## 2. PGN 格式详解

### 2.1 PGN 标签
```
[Event "Chinese Chess Game"]
[Date "2026.03.02"]
[Site "Local Game"]
[White "Red Player"]
[Black "Black Player"]
[Result "*"]
[Variant "Chinese Chess"]
[FEN "4rk3/3P5/4bP3/9/9/9/8P/1p2p2C1/3p1p3/4K1RR1 w - - 0 1"]
```

### 2.2 走法记录格式
```
1. 炮二平五 马８进７
2. 马二进三 车９平８
3. 车一平二 炮８进４
```

### 2.3 结果格式
- 红胜：`1-0`
- 黑胜：`0-1`
- 和棋：`1/2-1/2`
- 未结束：`*`

## 3. 走法字符串生成

### 3.1 调用 `generateMoveString` 方法
- 方法签名：`private String generateMoveString(int pieceType, Pos fromPos, Pos toPos, boolean isRed)`
- 生成规则：
  1. 计算起始列号（红方从右到左计数，黑方从左到右计数）
  2. 确定移动类型（进、退、平）
  3. 计算目标位置
  4. 获取棋子名称
  5. 生成走法字符串
  6. 黑方走法使用全角数字

## 4. 特殊标记处理

### 4.1 前/后/中标记处理
- **适用场景**：当同一列有多个相同的棋子时（如两马、两炮、两车等）

- **前/后判断逻辑**：
  - **红方**：
    - 前：离红方底线近（y值较大）
    - 后：离红方底线远（y值较小）
  - **黑方**：
    - 前：离黑方底线近（y值较小）
    - 后：离黑方底线远（y值较大）

- **中标记处理**：
  - 当同一列有3个相同棋子时使用
  - 按 y 坐标排序
  - 选择中间位置的棋子

- **示例**：
  - 前马、后马
  - 前炮、后炮
  - 前车、后车
  - 前卒、中卒、后卒

### 4.2 数字标记处理
- **适用场景**：当同一列有多个兵/卒时

- **处理逻辑**：
  - 按相对己方的顺序排序
  - 红方：从己方底线开始（y值大的在前）
  - 黑方：从己方底线开始（y值小的在前）
  - 选择对应索引的棋子，使用中文数字作为前缀

- **示例**：
  - 一兵、二兵、三兵、四兵、五兵
  - 一卒、二卒、三卒、四卒、五卒

### 4.3 实现细节
1. **收集同一列的相同棋子**：遍历棋盘，收集同一列中相同类型的棋子
2. **排序棋子**：根据y坐标排序
3. **添加前缀**：
   - 兵/卒：使用数字前缀（一、二、三、四、五）
   - 其他棋子：使用前后中前缀
4. **生成走法字符串**：将前缀与棋子名称、移动信息组合
5. **黑方走法处理**：自动转换为全角数字格式

## 5. 保存位置
- 使用外部存储空间：`context.getExternalFilesDir(null)`
- 目录：`/Android/data/top.nones.chessgame/files/`
- 文件扩展名：`.pgn`

## 6. 加载流程
- `ChessNotation.loadFromFile` 方法
- 支持 `.pgn` 和 `.txt` 文件
- 读取 PGN 格式文件
- 解析标签和走法记录
- 生成 `ChessNotation` 对象

## 7. 注意事项
- 保存时会包含完整的走法记录
- 支持 FEN 格式，便于加载和分析
- 兼容标准 PGN 格式，可与其他象棋软件交互
- 保存的棋谱可以在棋谱管理界面查看和加载
