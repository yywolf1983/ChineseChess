# 象棋规则测试用例

## 测试目的
验证实现的象棋规则是否正确工作

## 测试环境
- 项目：ChineseChess
- 测试模式：双人对战模式

## 测试用例

### 测试用例1：单方长将
**测试步骤**：
1. 开始新游戏
2. 让红方连续将军黑方6次
3. 观察是否出现"红方长将，必须变着"的提示
4. 尝试不变着，观察是否判负

**预期结果**：
- 第6次将军时出现强制变着提示
- 如果不变着，红方被判负

### 测试用例2：单方长捉
**测试步骤**：
1. 开始新游戏
2. 让红方连续攻击黑方的同一棋子3次
3. 观察是否出现"红方长捉，必须变着"的提示
4. 尝试不变着，观察是否判负

**预期结果**：
- 第3次攻击时出现强制变着提示
- 如果不变着，红方被判负

### 测试用例3：双方长将
**测试步骤**：
1. 开始新游戏
2. 红方将军1次，黑方将军1次
3. 重复步骤2，直到双方各将军3次
4. 观察是否出现"双方长将，双方不变作和"的提示

**预期结果**：
- 双方各将军3次后出现和棋提示
- 同意和棋后游戏结束

### 测试用例4：双方长捉
**测试步骤**：
1. 开始新游戏
2. 红方攻击黑方棋子1次，黑方攻击红方棋子1次
3. 重复步骤2，直到双方各攻击2次
4. 观察是否出现"双方长捉，双方不变作和"的提示

**预期结果**：
- 双方各攻击2次后出现和棋提示
- 同意和棋后游戏结束

### 测试用例5：双方闲着
**测试步骤**：
1. 开始新游戏
2. 双方走闲棋（无将军、无攻击）
3. 连续10回合
4. 观察是否出现和棋提示

**预期结果**：
- 连续10回合闲棋后不会出现和棋提示（闲着和棋功能已移除）
- 游戏继续进行

### 测试用例6：一方禁止一方允许
**测试步骤**：
1. 开始新游戏
2. 让红方长将（连续将军6次）
3. 黑方走闲棋
4. 观察是否出现"红方禁止着法，必须变着，不变判负"的提示

**预期结果**：
- 红方长将时出现强制变着提示
- 如果红方不变着，红方被判负

### 测试用例7：三次重复局面
**测试步骤**：
1. 开始新游戏
2. 走成某一局面
3. 通过悔棋和走棋回到同一局面3次
4. 观察是否出现"三次重复局面，请变着"的提示

**预期结果**：
- 第3次重复局面时出现强制变着提示

### 测试用例8：30回合未吃子
**测试步骤**：
1. 开始新游戏
2. 双方走棋，但30回合内不吃子
3. 观察是否出现"双方30回合内未吃子，是否和棋？"的提示

**预期结果**：
- 30回合未吃子后出现和棋提示

### 测试用例9：双方无攻击性棋子
**测试步骤**：
1. 在摆棋模式下，移除所有攻击性棋子（车、马、炮、兵/卒）
2. 开始游戏
3. 观察是否出现"双方都无攻击性棋子，是否和棋？"的提示

**预期结果**：
- 游戏开始时或无攻击性棋子时出现和棋提示

## 测试注意事项

1. **长捉检测**：需要确保连续攻击的是同一棋子
2. **攻击意图判断**：系统会检测移动后是否可以攻击对方棋子
3. **计数器重置**：当攻击不同棋子或吃子时，长捉计数器会重置
4. **规则优先级**：强制变着规则优先于和棋规则
5. **冷却机制**：强制变着后有3回合冷却期，避免频繁提示

## 调试信息

如果测试失败，可以检查以下信息：

1. **日志输出**：查看Logcat中"Move"标签的日志
2. **计数器状态**：
   - `consecutiveCheckRed/Black`：连续将军计数
   - `consecutiveAttackRed/Black`：连续攻击计数
   - `lastAttackedPiecePos/Type`：上次攻击的棋子信息
3. **规则类型**：`getViolatedRuleType()`返回的规则类型

## 已知限制

1. **长捉检测阈值**：可能需要根据实际对局调整（当前为3次）
2. **攻击意图判断**：基于棋子移动后的可能攻击目标，可能有误判
3. **双方闲着检测**：基于10回合无攻击，可能需要调整阈值
