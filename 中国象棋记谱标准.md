# 中国象棋记谱标准

## 基本规则

1. **记谱方法**：通常采用四字命名法，如"炮八平五"、"马二进三"等。

2. **记谱格式**：
   - 第1字：棋子的名称，如"马"、"车"、"炮"等。
   - 第2字：棋子所在纵线的数码，红方用大写数字（一至九），黑方用阿拉伯数字（1至9）。
   - 第3字：棋子移动的方向，横走用"平"，向前走用"进"，向后走用"退"。
   - 第4字：棋子进退的格数，或者到达纵线的数码。

3. **棋盘布局**：
   - 棋盘分为九路，红方从右至左为1至9路，黑方从左至右为1至9路。

## 坐标系统

### 棋盘坐标
- 棋盘横向（列）坐标：0-8，从左到右
- 棋盘纵向（行）坐标：0-9，从下到上
- 左下角为(0,0)，右下角为(8,0)
- 左上角为(0,9)，右上角为(8,9)

### 红方和黑方的计数方向
- 红方：从右到左计数，右为1路，左为9路
- 黑方：从左到右计数，左为1路，右为9路

### 坐标与记谱的对应关系
- 红方：记谱列号 = 9 - 棋盘x坐标
- 黑方：记谱列号 = 棋盘x坐标 + 1

   - 当一方有3个以上名称相同的棋子位于同一纵线时，需要用"前"、"中"、"后"来加以区别。
   - 对于兵（卒），当同一列有多个时：
     - 2个兵：使用"前兵"、"后兵"来区别
     - 3个兵：使用"前兵"、"中兵"、"后兵"来区别
     - 4个兵：使用"前兵"、"二兵"、"三兵"、"四兵"来区别
     - 5个兵：使用"前兵"、"二兵"、"三兵"、"四兵"、"五兵"来区别

## 兵的特殊规则

1. **移动方向**：
   - 兵过河前：只能向前走，用"进"表示
   - 兵过河后：可以向前、向左、向右走，用"进"或"平"表示
   - 兵永远不能后退，所以没有"退"的情况

2. **记谱示例**：
   - 正常情况：兵三进一（红兵从第三列向前走一步）
   - 过河平走：兵五平六（红兵从第五列平移到第六列）
   - 特殊情况：前兵进一（同一列最前面的红兵向前走一步）
   - 黑方记谱：卒3进1（黑卒从第3列向前走一步）

## 示例

- "炮二平五"：红炮从纵线二平移到纵线五。
- "马8进7"：黑马从纵线8向前走到纵线7。
- "车2退3"：黑车沿纵线2向后移动3格。
- "前马退六"：前面的红马退到直线六。
- "后炮平4"：后面的黑炮平移到直线4。
- "前兵进一"：同一列最前面的红兵向前走一步。
- "卒3平4"：黑卒从第3列平移到第4列。

## 注意事项

1. 红方使用大写数字（一至九），黑方使用阿拉伯数字（1至9）。
2. 横走用"平"，向前走用"进"，向后走用"退"。
3. 对于同一纵线上的多个相同棋子，使用"前"、"中"、"后"来区别。
4. 对于兵（卒），使用"一兵"、"二兵"、"三兵"、"四兵"、"五兵"来区别。

## 打谱软件标准操作规范

### 基本操作
1. **加载棋谱**：支持PGN格式的棋谱文件加载
2. **保存棋谱**：将当前对局保存为PGN格式
3. **上一步**：回退到上一步走法
4. **下一步**：前进到下一步走法
5. **开始**：回到棋谱的起始位置
6. **结束**：跳转到棋谱的最后一步

### 操作流程
1. **打开棋谱**：选择PGN文件，系统自动解析并显示初始局面
2. **浏览棋谱**：使用上一步/下一步按钮逐着浏览，或直接拖动进度条快速定位
3. **分析走法**：在每一步查看当前局面，分析双方走法
4. **保存修改**：如果对棋谱进行了修改，可以重新保存

## PGN格式说明

### 标签部分
```
[Game "Chinese Chess"]
[Event ""]
[Round ""]
[Date "2026-03-02"]
[Site ""]
[Red "Red Player"]
[Black "Black Player"]
[Result "*"]
[FEN "rnbakabnr/9/1c7/p1p1C1p1p/9/9/c1P3P1P/4C4/9/RNBAKABNR b - - 0 1"]
```

### 走法记录部分
```
1. 炮二平四 卒5平6
2. 兵四进一 将6进1
3. 车三进八 将6退1
```

### 注释格式
- 行内注释：`炮二平四 {#0,0#}`
- 块注释：`{#1,1#}`

## 打谱软件实现要点

1. **棋谱解析**：正确解析PGN格式，处理标签和走法记录
2. **局面管理**：维护当前局面状态，支持前进和后退操作
3. **走法验证**：确保走法符合象棋规则
4. **用户界面**：提供直观的操作按钮和进度显示
5. **错误处理**：优雅处理无效棋谱和异常情况
